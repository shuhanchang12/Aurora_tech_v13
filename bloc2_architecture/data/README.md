# 📊 Data Architecture Validation Samples

This directory contains validation sample datasets representing the three heterogeneous data sources integrated into the Star Schema warehouse. These are 10-row samples designed for schema verification and validation of ETL mapping logic.

## 1. `fx_sample.csv` — Foreign Exchange Rate Baseline

A 10-day snapshot of EUR/USD and EUR/TWD exchange rates (June 1–12, 2026), sourced from the [Frankfurter API](https://api.frankfurter.app/) ([source code](https://github.com/lineofflight/frankfurter)).

| Column | Type | Description |
|:---|:---|:---|
| `base_currency` | VARCHAR(3) | Source currency (always EUR) |
| `target_currency` | VARCHAR(3) | Target currency (USD or TWD) |
| `exchange_rate` | DECIMAL(10,4) | The exchange rate at retrieval time |
| `retrieved_at` | TIMESTAMP | When the rate was captured (daily 16:00 CET) |

> **Schema Ingestion:** Columns align with `staging.fx_rates_raw` in [`bloc3_pipelines/sql/create_tables.sql`](../../../bloc3_pipelines/sql/create_tables.sql).

## 2. `laptop_price_sample.csv` — Laptop Specifications & Pricing Matrix

A schema verification sample containing laptop specifications, attributes, and base pricing in EUR, sourced from public laptop price listings.

| Column | Type | Description |
|:---|:---|:---|
| `laptop_ID` | INT | Unique identifier for the laptop model |
| `Company` | VARCHAR | Manufacturer brand (Apple, HP, Asus, etc.) |
| `Product` | VARCHAR | Model line name |
| `TypeName` | VARCHAR | Category (Ultrabook, Notebook, Gaming, etc.) |
| `Inches` | DECIMAL | Screen size in inches |
| `ScreenResolution` | VARCHAR | Display resolution and panel technology |
| `Cpu` | VARCHAR | CPU specification detail |
| `Ram` | VARCHAR | Memory capacity (e.g., "8GB") |
| `Memory` | VARCHAR | Storage drive types and sizes |
| `Gpu` | VARCHAR | Graphics hardware |
| `OpSys` | VARCHAR | Operating system pre-installed |
| `Weight` | VARCHAR | Weight of the laptop (e.g., "1.37kg") |
| `Price_euros` | DECIMAL | Base retail price in Euros |

> **Schema Ingestion:** Columns align with `staging.laptop_prices_raw` in [`bloc3_pipelines/sql/create_tables.sql`](../../../bloc3_pipelines/sql/create_tables.sql).

## 3. `electronic_sales_sample.csv` — Sales & Supply Chain Logistics Transactions

A transaction log sample tracking international order logistics, customer details, and total purchase prices.

| Column | Type | Description |
|:---|:---|:---|
| `Customer ID` | INT | Unique identifier for the customer |
| `Age` | INT | Age of the customer |
| `Gender` | VARCHAR | Gender of the customer |
| `Loyalty Member` | VARCHAR | Yes/No flag for membership |
| `Product Type` | VARCHAR | Product class category (Smartphone, Laptop, etc.) |
| `SKU` | VARCHAR | Unique product identifier code |
| `Rating` | INT | Customer rating of the transaction (1-5) |
| `Order Status` | VARCHAR | Status (Completed, Cancelled, etc.) |
| `Payment Method` | VARCHAR | Payment type used |
| `Total Price` | DECIMAL | Total revenue generated by the transaction |
| `Unit Price` | DECIMAL | Base price per unit of the item |
| `Quantity` | INT | Quantity of items purchased |
| `Purchase Date` | DATE | Date of transaction |
| `Shipping Type` | VARCHAR | Shipping tier (Standard, Express, Overnight) |
| `Add-ons Purchased` | VARCHAR | List of optional accessories/add-ons purchased |

> **Schema Ingestion:** Columns align with `staging.electronic_sales_raw` in [`bloc3_pipelines/sql/create_tables.sql`](../../../bloc3_pipelines/sql/create_tables.sql).

### Star Schema Integration Design

These three sources are ingested into the staging area, cleaned, and integrated into the Star Schema dimensions and fact tables:

```
[ heterogeneous sources ]       [ staging tables ]                  [ star schema DWH ]

fx_sample.csv             ──►  staging.fx_rates_raw          ──┐
laptop_price_sample.csv   ──►  staging.laptop_prices_raw     ──┼─►  dwh.fact_chromebook_margin_risk
electronic_sales_sample.csv ──► staging.electronic_sales_raw ──┘          ├── dwh.dim_date (via Date)
                                                                          ├── dwh.dim_hardware_product (via SKU)
                                                                          └── dwh.dim_chromebook_vendor (via Vendor)
```

This integration enables the DWH to monitor daily exchange rate exposure against supply chain shipping delays and product cost fluctuations to manage real-time gross margin risks.
