6. Model Evaluation Strategy

Metrics Selection

  • Metric: F1 Score prioritized over Accuracy.

7. Model Serialization

Storage strategy

8. Architecture: API Development

Deployment Layer

  • Framework: FastAPI (for async robustness and validation).
  • Endpoint: /predict-margin-risk.
  • Validation via Pydantic payload models.
Slide 9 / 16

9. Containerization

Docker Configuration

  • Dockerization using python:3.10-slim.
  • Encapsulates execution environment entirely to prevent dev-prod skew.
Slide 10 / 16

10. CI / CD Pipeline

Automation pipeline

  • GitHub Actions (.github/workflows/mlops-ci.yml).
  • Fires on branch push.
  • Runs PyTest to ensure Inference API compatibility.
Slide 11 / 16

11. Kubernetes Orchestration

K8s Strategy

  • Deployment YAML specifies 2 Replicas for failover.
  • Integrated Liveness Probe over /health.
Slide 12 / 16

12. Production Monitoring & Drift

Monitoring Setup

  • Script: monitoring/monitor.py.
  • Measures Euclidean distance / KS test vs baseline distributions.
Slide 13 / 16

13. Automated Retraining

Self-healing system

  • When P-value falls below 0.05, monitor script fires run_retrain.py.
  • Model automatically fits on updated PostgreSQL tuples.
Slide 14 / 16

14. Production Cutover

Quality gates

  • After retrain success, QA (PyTest) must pass before swapping endpoints.
  • Maintains stability under automatic retraining environments.
